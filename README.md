# Care Nova AI Data Folder

This folder contains the safe GitHub-visible data structure for Care Nova AI.

Included in GitHub:

- `offline-medical-db.json`: curated offline seed knowledge used by the browser and local model runtime.
- `offline-clinical-repository.json`: expanded offline repository records for disease, medicine, lab, imaging, procedure, safety, records, insurance, and prevention retrieval.
- `offline-knowledge-index.json`: dependency-free local lexical/vector-style retrieval index generated from the offline repository.
- `offline-repository-manifest.json`: source-family, update, licensing, governance, and maintenance workflow metadata.
- `offline-source-packs/*.json`: modular local knowledge packs that are merged into the repository during rebuild.
- `.gitkeep` files inside runtime folders so the complete data layout is visible after upload.

Rebuild the offline repository:

```powershell
.\node.cmd scripts\build-offline-repository.js
```

The generated repository is designed for offline search and safe education. It does not bundle copyrighted textbooks, proprietary clinical guidelines, diagnostic rules, prescription dosing, or treatment orders. Those can only be imported after license approval, PHI removal, clinician review, validation, and rollback preparation.

Source-pack workflow:

- Add or update curated local records in `data/offline-source-packs/`.
- Keep those records focused on symptom review, safety boundaries, preparation, records, workflows, and patient-friendly education.
- Rebuild the repository so packs are normalized, enriched, indexed, and included in the manifest.

Kept local only:

- `memory/`: patient memory generated by the local server.
- `records/`: patient records saved by the local server.
- `graph/`: patient knowledge graph facts.
- `training/`: local feedback and calibration state.
- `external/`: optional cached online-source references.
- `audit/`: local operational audit events for enterprise review.
- `onedrive-mirror/`: local OneDrive mirror copies.

Private runtime files in those folders are intentionally excluded from GitHub. The app recreates them automatically during local use.

Enterprise runtime controls:

- `CARE_NOVA_REQUIRE_ADMIN_FOR_MUTATIONS` can require an admin token before protected mutation routes change local state.
- `CARE_NOVA_ADMIN_AUTH_REQUIRED` plus `CARE_NOVA_ADMIN_SESSION_SECRET` can protect enterprise-only review routes such as policy, audit, backup, integrity, and runtime metrics.
- `CARE_NOVA_MAINTENANCE_MODE` can pause protected mutation routes during operational work.
- `CARE_NOVA_READ_ONLY_MODE` can freeze protected mutation routes while leaving read access available.
